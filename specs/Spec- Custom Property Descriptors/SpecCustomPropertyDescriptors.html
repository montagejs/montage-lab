Introduction

Montage has started down the path of allowing, custom property descriptors.  As of this writing it supports the following extensions, distinct, modify and serializable, with code in the works for a needsDraw extension.  Currently the code for handling these extensions is all defined in Montage.defineProperty.  To increase maintainability and separation of concerns Montage should support a way to register custom property descriptors which does not require changing Montage.defineProperty every time the framework wishes to support a new custom descriptor.  Furthermore as it heads further down the path of supporting custom descriptors which are only applicable to objects of a certain type the code for handling those descriptors should be in context of the objects that care.  See the needsDraw custom descriptor spec for an example.

Functional Description

To support adding custom property descriptors a new API will be provided on the Montage core object, defineCustomPropertyDescriptor.  This method will take two parameters, the name of the custom descriptor and the function that should be called when the custom descriptor is encountered.  The callback function should take two parameters, the name of the property that this custom descriptor is being applied to and the descriptor object itself.  The callback method is not expected to return anything and all return values will be ignored.

API

defineCustomPropertyDescriptor(name, funktion)

Concerns

  1. Depending on how the custom descriptor is implemented it can cause unexpected behavior with hasOwnProperty and getOwnPropertyDescriptor
  2. getOwnPropertyDescriptor will not return any of the custom property descriptor values
  3. Order of calling custom descriptor functions, cases of where the custom descriptor rewrites the property into two separate properties, needsDraw and distinct, how do you know that the two are mutually exclusive?
  1. Correction some of the custom descriptors are mutually exclusive, i.e. needsDraw and distinct, but modify and serializable are not
  1. Should you be able to unregister a custom descriptor?  What would that even mean?
