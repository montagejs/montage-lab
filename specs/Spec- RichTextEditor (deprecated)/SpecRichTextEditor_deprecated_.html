The RichText editor is composed of 3 components: the editor, the sanitizer and the resizer.

The editor is basically a wrapper around a content editable div which allow to do rich editing. You can execute commands such as bold, underline and italic, as well inserting bullet list or manipulating font and colors. The editor does not provide a toolbar, however you can defined you own use binding  to execute a commands or display a state.

The sanitizer is a component in charge of sanitizing the content in order to make sure that neither css or javascript might interfere with the rest of the page. The Resizer is a component that allow the user to resize images. While the editor provides it's own sanitizer and resized, the consumer can replace or extend them as needed.

Montage RichText Editor Component

Variables

value        string - content in HTML

textValue        string - plain text version of the content (won't work with FF for now as it uses innerText)

states        array objects (readonly) - state of supported styles at the current selection or  caret position

allowDrop        boolean - specify if drops are allowed (drag & drop)

actions        array string (readonly) - list of supported actions by the rich text editor

enabledActions        array string - list of action the user is allowed to perform, by default all actions are enabled

sanitizer        Object - object the consumer can replace which is in charge or sanitizing the content. By default uses the Sanitizer provided by the editor (see interface below)

resized[a]                        Object - object the consumer can replace which is allow the user to resize images. By default uses the Resizer provided by the editor (see interface below)

delegate        Object - delegate object  (see delegate callbacks bellow)

Methods

updateStates()        refresh the states, returns states. State are not automatically updated, you must call this method periodically or when receiving a editorSelect event.

focus()        set focus on editor

doAction(action, value)[b]        execute an action

Events

editorChange        posted when the content of the editor has changed, this is not done in real time

editorSelect        posted when the selection change (this is not done in real time)

Delegate Callbacks

drop(editor, data, contentType)        the delegate provide a change for the consumer to handle a drop itself, refuse the drop, accept the drop or change the drop data

fileDrop(editor, data, contentType)        the delegate provide a change for the consumer to handle a file drop itself, refuse the drop or accept it

paste(editor, file, data)        the delegate provide a change for the consumer to handle a paste itself, refuse the paste, accept the paste or change the paste data.

filePaste(editor,  file, data)        the delegate provide a change for the consumer to handle a file paste itself, refuse the paste or accept the paste.

Montage RichTextEditor Sanitizer

willSetValue(value, identifier)        called when the editor's value is about to be set, you can alter the value before returning it. Typically you can to scope any css rules to prevent interring with the rest of the page. The identifier is the unique id of the editor instance

didGetValue(value, identifier)        called when getting the editor's value. can you undo any changes performed by willSetValue(). Returns the altered value.

willInsertHTMLData[c](data, identifier)        called during a drop or paste of HTML fragment. Ideally you want to scope css and remove scripts. Returns the altered value.

Montage RichTextEditor Resizer

initialize(editor)        Initialization of the resizer

show(element)        Add a resized around the element. This is called during a draw, therefore you can manipulate the DOM directly. Note: only one element can show a resizer at a time, you are responsible to remove a previous resizer.

hide()        Remove the current resizer. Called during a draw.

cleanup(contentNode)        Return a copy of the content node without the resizer. This must not affect the currently displayed resizer, therefore  you are responsible to clone the node if needed before altering it. Returns a clean copy of the node.
