Autocomplete[a] Component

An Autocomplete Textfield extends the Textfield component to support suggestions for the user to pick values. The HTML markup for the Autocomplete is the same as the standard HTML5 markup (<input>). Wrapping the <input> HTMLElement as a Montage Autocomplete component adds Data Binding support for all writable attributes of this element and allows the Developer to provide suggestions to the user based on the entered value.

It supports all (most?) standard attributes of the <input> HTMLElement as specified here - http://www.w3.org/TR/html5/the-input-element.html#the-input-element

https://developer.mozilla.org/en/HTML/Element/Input

Properties

Properties specific to Autocomplete

delegate[b]

The Delegate object provides an extension point for a Developer using the Autocomplete component to suggest values to the end user based on the currently entered value.

The Delegate is an instance of a Montage object with one of the following methods:

Option #1: Implement[c] method with the format: <identifier>ShouldGetSuggestions() where <identifier> is the ID of the Autocomplete component.

Eg: statesShouldGetSuggestions(autocomplete, searchTerm) where ‘states’ is the “identifier” of the Autocomplete component.

This is useful when the same instance must provide suggestions for multiple Autocomplete components.

Option #2: Implement a method: shouldGetSuggestions(autocomplete, searchTerm). In this case, it is upto the Developer to return the appropriate suggestions based on the autocomplete instance that is passed in.

Parameters:

autocomplete: The component instance requesting the suggestions

searchTerm[d]: The user-entered search term for which suggestions are being sought from the delegate.

It is upto the delegate to fetch the suggestions from an in-memory/ local or remote data source (via XHR/JSONP etc). Once the response has been received, the delegate must set the “suggestions” property on the autocomplete component. Upon receiving the suggestions, the Autocomplete component allows the user to select a value from the suggestions.

Here is an example:

stateShouldGetSuggestions: {

        value: function(autocomplete, searchTerm) {

            var results = [];

            if(searchTerm) {

                var term = searchTerm.toLowerCase();

                if(this._cachedStates && this._cachedStates[term]) {

                    results = this._cachedStates[term];

                } else {

                    results = states.filter(function(item) {

                        // @todo - memoize

                        return (item.name.toLowerCase().indexOf(term) >= 0 || item.code.indexOf(term) >= 0);

                    });

                    if(this._cachedStates == null) {

                        this._cachedStates = {};

                    }

                    this._cachedStates[term] = results;

                }                

            }

            autocomplete.suggestions = results.map(function(item) {

                return item.name;

            });                

        }

    },

delay[e]

Time in milliseconds that the component should wait before providing suggestions based on the current value.

Type: Integer

Default: 500 (milliseconds)

minLength[f]

The number of characters the user must enter before the component provides suggestions.

Type: Integer

Default: 2

separator

The character that is used as a separator between terms.

Type: String

Default: ‘,’

Notes

- Combobox mode for Autocomplete (single and multiple)

- Validate entered value with delegate in combobox mode

- Is Combobox another component that uses Autocomplete?

- propertyPath -> delegate returns Object array. Autocomplete uses Array.getProperty(propertyPath)

- There is a cost to using a new Array from the delegate. Instead, use the same array and change the contents.

"autocomplete" might be mistaken with the attribute in forms like: <input type="text" autocomplete="off">. Which basically is just to turn on/off the browser's "auto-fill".

Based on the functionality.. it's more a polyfill for the native <datalist> https://developer.mozilla.org/en/HTML/Element/datalist. Browser-support isn't that great yet.. http://caniuse.com/#search=datalist but I guess should also come to WebKit soon. So the name would be "datalist.reel".

But it's in the same category as the skeleton/input-range. And because the Skeleton theme in montage/lab/skeleton is not ready yet for real use.. I would suggest that we add a "skeleton" folder back like we used to have it: "montage/ui/skeleton/".. it's where all our polyfills can be for now. So:

montage/ui/skeleton/input-range.reel/

montage/ui/skeleton/datalist.reel/

Or add a folder called "polyfills":

montage/ui/polyfills/input-range.reel/

montage/ui/polyfills/datalist.reel/

Both are probably still not final, (have a different folder structure in mind when we add theming).
