Popup Component

Montage provides a number of different ways to overlay information screens via Popups.

Properties[a]

anchor[b]

Type: HTMLElement or Component

If provided, the popup will be positioned below the anchor element. There is no support currently to position it above/left/right of the anchor. However, custom positioning is possible by using a delegate. See “delegate” below.

If an anchor is not provided, the position property is used if provided. If no positioning support is provided, the Popup is displayed at the center of the screen by default.

position[c]

Type: Object

Optional. Object with “top”, “left”, “bottolm” and “right” properties to specify the position of the popup.

delegate

Type: Montage Object

Optional. The only delegate method currently supported is willPositionPopup. This method must return an object with any of the properties “top”, “left”, “bottom” or “right”. If the value of a property is a number then it is the number of pixels[d], otherwise you can use a string with any CSS unit, such as %. defaultPosition is an object with “top”, “left”, “bottom” and “right” properties.

willPositionPopup: {

       value: function(popup, defaultPosition[e]) {

                return {top: 10, left: 10}

        }

}

content

Type: Montage Component

The Montage component that must be shown inside the Popup.

modal[f]

Type: boolean

Default: false

Optional. If set to true, the popup will be rendered as a Modal. A modal popup is rendered with a mask underneath the popup. A non-modal popup is hidden when the user presses the Esc key, or clicks outside of the popup. The developer is responsible for hiding a modal popup. Popup doesn’t autoHide.

autoHide[g]

Type: number

Optional. If provided, the popup will be automatically hidden after the specified # of milliseconds. This property has no effect on modal popups.

API

show() - Show the Popup

hide() - Hide the Popup

API Usage

1) Show a Montage component in a Popup.

Lets say we have a Montage component that is defined thus:

"menubookmark1": {

        "name": "MenuBookmark",

        "properties": {

            "label": "The New York Times"

        }

    },

This is a Menu that must be displayed as an overlay upon the user clicking a button. The button action is wired as follows:

"showBookmarkMenu": {

        "module": "montage/ui/button.reel",

        "name": "Button",

        "properties": {

            "element": {"#": "action-show-bookmark-menu"},

            "identifier": "showBookmarkMenu"

        },

        "listeners": [

            {

                "type": "action",

                "listener": {"@": "owner"}

            }

        ]

    },

The handler for the “showBookmarkMenu” action is as follows:

bookmarkMenu: {value: null}, // defined in the serialization

 _bookmarkMenuPopup: {value: null},

handleShowBookmarkMenuAction: {

        value: function(evt) {

            var popup = this._bookmarkMenuPopup;

            if(!popup) {

        // create a Popup instance as we don’t have one yet.

                popup = Popup.create();

// Mark it as a modal

                popup.modal = true;[h]

        // set the content of the Popup as the Bookmark Menu component

        // save this popup instance for later use

                this._bookmarkMenuPopup = popup;

        // add a listener to handle ‘Add Bookmark’ action

    this.bookmarkMenu.addEventListener('addbookmark', this);

            }

2) Using the Montage provided popups - Alert/Confirm/Notifier

Montage comes with some Popups such as Alert, Confirm and Notifier. The following examples show how to use them.

Add the require’s to your code:

Alert = require("montage/ui/popup/alert.reel").Alert,

Confirm = require("montage/ui/popup/confirm.reel").Confirm,

Notifier = require("montage/ui/popup/notifier.reel").Notifier;

Alert.show(message, <okCallback>)

Alert.show('We were unable to process your request. Please try again later.', function() {

self.log("User clicked OK on the alert dialog");

});

Confirm.show(message, <okCallback>, <cancelCallback>)

if(Confirm.show('Are you sure?', function() {

self.log('User clicked Restart');

               self._criticalErrorPopup.hide();

}));

* Show the Notifier popup with the specified message.

* @param message

* @param timeout (optional). If null, a default timeout of is used

3) Use a Popup Delegate to position the Popup using custom logic

The Montage Popup Component supports a couple of standard defaults to position the Popup:

- Set the “position” to specify the exact location

- Set an “anchor” to position the popup below the anchor element.

- Or default to the center of the screen

However, there are situations where the Application needs more control over where the Popup is shown. This is achieved using the Popup delegate. To use the delegate, define a Montage object as follows:

var delegate = Montage.create(Montage, {

        willPositionPopup: {

                value: function(popup, defaultPosition) {

                        return {top: 1, right: 5};

                }

        }

});

Set this Object as the delegate of the Popup:

var myPopup = Popup.create();

myPopup.anchor = someButton;

myPopup.delegate = delegate;
