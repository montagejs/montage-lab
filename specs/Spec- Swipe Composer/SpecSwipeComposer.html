Introduction

The swipe composer is responsible for aggregating touch events[a] on touch enabled devices into the swipe[b] event if they meet the criteria for a swipe gesture.

Functional Description

Developers can add listeners for the swipe events from swipe composers that are added to their components.  There is one event fired by the swipe composer, swipe.  Events are only fired for pages viewed on touch enabled devices.

Events

swipe

Fired when touchstart, touchmove and touchend events are analyzed and detected to fit the criteria for a swipe

API

SwipeEvent

The swipe event is a custom event containing the following properties

type

swipe | swipemove | swipecancel

direction

DIAGONAL | RIGHT | LEFT | DOWN | UP

angle[c]

Number: the angle of ?

velocity

Number: the velocity of the swipe as calculated by

startX[d]

Number: the value of x when the swipe started

startY

Number: the value of y when the swipe started

deltaX

Number: the change in X between the swipe start and swipe end

deltaY

Number: the change in Y between the swipe start and swipe end

Examples

var Component = require("ui/component").Component,

   SwipeComposer = require("ui/composer/swipe-composer").SwipeComposer;

var MyComponent = Montage.create(Component, {

   // ...

   prepareForDraw: {

       value: function() {

           var swipeComposer = SwipeComposer.create();

           this.addComposer(swipeComposer);

           swipeComposer.addEventListener("swipe", this, false);

       }

   },

   handleSwipe: {

       value: function(event) {

           // do something

       }

   }

   // ...

});
