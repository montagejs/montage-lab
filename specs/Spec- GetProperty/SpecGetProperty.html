Introduction

getProperty is a function implemented at Object.prototype level that is meant to access properties of an object by augmenting the current ECMAScript property access capabilities. Other prototypes may implement custom implementations of getProperty with unique capabilities for reading properties.

API

<object>.getProperty(path, [visitorCallback])

The default implementation allows the developer to access an object’s property just like ECMAScript does:

object["propertyName"] <=> object.getProperty("propertyName");

object.propertyName => object.getProperty("propertyName");

The usefulness of getProperty starts when we need to go deeper.[1]

Traversing an object graph can easily involve a lot of boilerplate code just to make sure that each arc in the graph is an actual link to another object and not just a dead end (undefined/null), example:

function lambda(object) {

        var y = object.x.y;

}

lambda({x: {y: 2}});

The problem starts if the chain is broken and one of the properties is undefined or null:

lambda({});

This execution will result in can’t read property "y" from undefined.

The correct code would have to be:

function lambda(object) {

        var y;

        if (typeof object.x != null) {

                y = object.x.y;

        }

}

Or,

var empty = {};

function lambda(object) {

    return ((object || empty).x || empty).y;
}

For deeper paths a chain of if’s would have been necessary.

With getProperty this access is as simple as:

function lambda(object) {

        var y = object.getProperty("x.y");

}

Parameters

Path

[string]

A property to access on the target object. A property path can be formed by chaining properties with the dot (.) operator. The first property is resolved on the target object. Subsequent properties in the path are visited in order rooted at the result of resolving the previous property.

Visitor

[null|function(target, propertyName, result, remainingPath)]

getProperty also accepts a function as a second argument that will act as a visitor to each object visited during the property path traversal.

        The visitor’s signature is: function(target, propertyName, result, remainingPath):

target: the target of propertyName

propertyName: the property name being visited

result: the value being visited target[propertyName]

remainingPath: the rest of the path in relation to this result and the path given to getProperty.

Example:

var object = {x: {y: 2}};

object.getProperty("x.y", function visitor(target, propertyName, result, remainingPath) {...});

In this execution the visitor function will be called with the following arguments:

visitor(object, "x", object.x, "y");

visitor(object.x, "y", object.x.y, null);

Arrays

Arrays have their own implementation of getProperty. To access a specific index of an array the index is used just like any other property name.

Examples:

array.getProperty("0");

object.getProperty("array.0");

object.getProperty("array.0.x");

The main reason that Array implements its own getProperty is to provide more functionality than Object.prototype.getProperty.

Instead of just retrieving a property from a specific element in the array it is also possible to get that property from ALL elements in the array.

Example:

var array = [{x: 1}, {x: 2}, {x: 3}, {x: 6, y:1}, {y: 1, x: 8}];

array.getProperty("x") === [1, 2, 3, 6, 8];

Visitor Example:

var object = {array: [{x: 1}, {x: 2}, {x: 3}]};

array.getProperty("array.x", function visitor(target, propertyName, result, remainingPath){...});

The visitor needs to visit all objects visited during the path traversal:

visitor(object, "array", object.array, "x");

visitor(object.array, null, undefined, "x");

visitor(object.array[0], "x", object.array[0].x, null);

visitor(object.array[1], "x", object.array[1].x, null);

visitor(object.array[2], "x", object.array[2].x, null);

Even though (object.array, {"0","1","2"}) is visited during the traversal,  the property path "array.foo" doesn’t really depend on these specific indexes, it depends on the array as a whole, hence visiting the array with no specific property name.

Collection Property Path Functions

Consider the following scenario when reviewing collection property path functions:

var alice = {“name”: “Alice”, “salary”: 100};

var bob = {“name”: “Bob”, “salary”: 200};

…

var frank = {“name”: “Frank”, “salary”: 600};

var engineering = {“name”: “Engineering”, “employees”: [alice, bob, carol]};

var marketing = {“name”: “Marketing”, “employees”: [david, eve, frank]};

var departments = [engineering, marketing];

var company = {“departments”: departments};

unique()

Remove duplicates from the result of the preceding property path:

        var georgia = {“name”: “Georgia”};

        engineering.employees.push(georgia);

        marketing.employees.push(georgia);

        var georgia2= {“name”: “Georgia”};

        marketing.employees.push(georgia2);

company.getProperty(“departments.employees.name.unique()”);

=> ['Alice', 'Bob', 'Carol', 'Georgia', 'David', 'Eve', 'Frank']

The above example get a list of all the employee names and then removes duplicate names. Note that this differs from the following example which will get a list of the employees with duplicates removed, and then get their names. This means you may end up with duplicates in the result if multiple employees have the same name.

company.getProperty(“departments.employees.unique().name”);

=> ['Alice', 'Bob', 'Carol', 'Georgia', 'David', 'Eve', 'Frank', 'Georgia']

Invalid (propertyPath functions only valid on collections)

company.getProperty(“unique().departments.employees.name”);

obj.getProperty(“a.b.count()”) => 4

By default duplicates are not removed at all:

company.getProperty(“departments.employees.name”);

=> ['Alice', 'Bob', 'Carol', 'Georgia', 'David', 'Eve', 'Frank', 'Georgia', 'Georgia']

Rolling your own

It is possible to implement a getProperty function on any kind of object as long as the visitor rules are respected, it is part of the API.

        The visitor is the underlying function used by the ChangeNotification API to know which objects are dependencies of a specific property path.
